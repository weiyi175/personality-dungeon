4) Fixed Execution Order

  1. E0：刷新介面輸出（建立同批次基準）
  2. E1：重現當前跨 block 狀態（確認失敗可再現）
  3. E2/E3/E4：只在 fail blocks 做 parameterized repair experiments
  4. E5：最佳 candidate 回填 4 blocks 並重跑 W2 + A3 做正式 closure

若 E2~E4 沒有任何 candidate 通過，結論維持 NO-GO，並以 negative result 形式記錄，不進行無界擴張掃描。

5) SDD Convergent Flow (可收斂執行規格)

步驟 執行規則 驗收輸出
固定契約（Control Lock） 固定同一組 block、同一 fairness window/tolerance、同一 baseline。
E1 視為對照組，不在後續修復掃描中變動。 		E1 四份 gate summary + W2 focus rows=4，作為唯一比較基準。
只打 fail blocks（Repair Scope Lock） 只針對 102..161 / 162..221 / 222..281 執行 E2→E3→E4。
不回頭改 E1 的 42..101，避免混入額外變因。 		每個機制族都要有 3 個 block 的對照可比結果。
候選淘汰規則（Hard Elimination） 候選必須在 3 個 fail blocks 同時滿足：
1) overall_pass=true
2) new_l1=0
3) fairness_fail_count=0
任一 block 不成立即淘汰該候選。 		candidate status: survive 或 eliminate。
回填四個 block（Closure Refill） 只將唯一存活候選回填至 4 blocks（含 42..101），然後重跑 W2 與 A3。 outputs/w2/block_risk_report_v1.json、outputs/track_a_protocol_regression_summary.json。
最終 GO 條件（Final Gate） 必須同時滿足：
1) w2_decision.block_robustness == "block_robust"
2) focus.mean_new_l1 == 0.0
3) a3_drift_detection.verdict == "stable"
4) track_a_protocol_regression_summary.overall_pass == true 		全部成立才可將 Event line 由 NO-GO 改為 GO。
停損規則（Stop-Loss） 若 E2/E3/E4 全掃完仍無候選同時通過 3 個 fail blocks， 當季收斂為 negative result，停止無界擴張。 決策維持 NO-GO，並更新 W4 決策文件。

6) Experiment Method (避免假改善)

方法原則 操作規範 決策意義
一次只改一個機制族 固定順序：先 E2（dispatch 時序）→ E3（reward modulation）→ E4（impact spread）。
每輪只動單一機制族，其他參數保持 lock。 		可把失敗原因定位到單一機制，避免混合因果。
先 gate，後次要指標 主判準僅看 gate 條件（overall_pass、new_l1、fairness）。
mean_s3、mean_gamma 只作 tie-break，不能推翻 gate 失敗。 		防止「看起來有改善」但實際不具重啟資格的假陽性。
同步保留負結果 每個淘汰候選都要記錄失敗子條件（例如 new_l1_pass=false 或 l1_pass=false）。 直接指向下一輪該改的機制，不做盲掃。

7.2 Run Order（收斂 + fail-fast）

  1. 先跑 E2 全部候選；若無存活候選，才進入 E3。
  2. 再跑 E3 全部候選；若無存活候選，才進入 E4。
  3. 再跑 E4 全部候選；若仍無存活候選，直接觸發 Stop-Loss。
  4. 任一候選若在 3 個 fail blocks 都存活，立即進入 E5 closure refill。

block 內建議順序：102..161 → 162..221 → 222..281。 候選一旦在任一 block 失敗，可立即標記 eliminate（可節省算力）。
