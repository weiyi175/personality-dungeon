W4 實驗藍圖

主線 80% 探索 20% Go/No-Go

W4 核心目標：完成月結回顧與 go/no-go 判定，明確決定是否維持主線、是否允許 Event line 重啟，並把決策寫入可追溯紀錄。

1. W4 目標與交付物

工作流 目標 W4 交付物
A3 月結回顧（主線 80%） 確認主線是否持續穩定且可進下一週期 monthly review v1
W4 go/no-go 判定（探索 20%） 決定 Event line 是否可重啟或維持凍結 go/no-go memo v1
決策可追溯化 把判定、證據、責任歸屬寫成固定紀錄 decision record v1

2. 範圍定義（In / Out）

2.1 In Scope

  • 使用 W3 已完成產物進行 W4 決策彙整與判讀。
  • 對主線穩定性、跨 block 穩健性與重啟守門條件進行結論化。
  • 輸出 go/no-go memo、月結回顧與決策紀錄。

2.2 Out of Scope

  • 不新增機制實作或 runtime 行為改動。
  • 不執行新的大規模機制掃參。
  • 不在 W4 直接啟動 Event line 正式重啟實驗。

4. 執行流程

4.1 主線 80%：A3 月結回顧

./venv/bin/python -m simulation.track_a_protocol_regression

月結判讀最少需覆蓋：

  • overall_pass 是否維持 true
  • 核心門檻：L1 <= 3、Healthy >= 42
  • 相對 W2/W3 的 drift 是否可接受

4.2 探索 20%：重啟條件評估（不實作）

W4 只做判定，不做重啟實作。至少檢查以下條件：

  1. 跨 block 穩健證據是否成立（非單 block 偶然通過）。
  2. new_l1 = 0 是否可被滿足。
  3. H-01 至 H-03 的 fail-fast 規則是否完整可執行。

4.3 決策紀錄化

對每一個 go/no-go 結論都需記錄：

  • decision_id、決策日期、決策人
  • 判定結果（go / no-go / defer）
  • 證據清單與風險備註
  • 下次覆核條件與時間點

5. W4 時程（五天版）

Day 主線 80% 探索 20% 當日輸出
D1 凍結 W3 證據輸入 建立 W4 判定框架 W4 source manifest
D2 主線月結指標快檢 彙整 H-01~H-03 可證偽條件 monthly review draft
D3 A3 月結 gate 判定 重啟守門條件對照 go/no-go draft
D4 風險與替代方案審視 decision record 草稿 decision record draft
D5 W4 結論定稿 下一週期建議 go/no-go memo v1 + monthly review v1 + decision record v1

6. 驗收標準（Definition of Done）

  1. go/no-go memo v1 有明確結論、理由、風險與下一步。
  2. monthly review v1 完整覆蓋 W1~W3 到 W4 的月結脈絡。
  3. decision record v1 至少包含 1 筆可追溯決策條目。
  4. Event line 判定符合重啟守門條件（跨 block 穩健且 new_l1=0）。
  5. W4 全程維持「先判定再實作」紀律。
