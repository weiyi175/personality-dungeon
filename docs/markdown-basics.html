Markdown 基本使用

Markdown 是一種輕量級標記語言，由 John Gruber 於 2004 年創建，目的是提供一種易於閱讀、易於撰寫的純文字格式，並且可以轉換為結構化的 HTML。 Markdown 的設計理念是讓文件在純文字狀態下也能保持良好的可讀性，同時又能輕鬆轉換為格式豐富的文件。

Markdown 的優點在於其簡潔的語法和廣泛的應用場景。無論是撰寫部落格文章、技術文檔、README 檔案，還是在各種支援 Markdown 的平台上進行寫作，Markdown 都能提供一致且高效的寫作體驗。

Markdown 的不同版本與擴展

隨著 Markdown 的發展，出現了多個版本和擴展，了解它們的差異有助於您選擇合適的語法：

版本 / 擴展 說明
原始 Markdown 創建者：John Gruber（2004 年）
特點：最基本的 Markdown 語法規範
包含功能：標題、粗體、斜體、連結、圖片、列表、引文、代碼、分隔線等核心功能
CommonMark 目的：標準化 Markdown 規範，消除原始規範中的歧義
特點：提供明確的解析規則和測試套件
擴展：添加了圍欄式代碼區塊等功能
GitHub Flavored Markdown (GFM) 基於：CommonMark 規範
擴展功能：表格語法、刪除線、任務列表、自動連結識別、語法高亮
其他擴展 Pandoc：上標、下標、定義列表等
MultiMarkdown：腳註、元數據、數學公式等
各編輯器擴展：高亮文字、表情符號、圖表等

主標題與副標

✅ 標準 Markdown 功能

本章介紹的標題語法是原始 Markdown 規範的核心功能，所有 Markdown 實現都支援。

在 Markdown 中，標題是文檔結構的重要組成部分，用於區分不同層級的內容和創建清晰的文檔層次結構。 Markdown 支援六個級別的標題，從 # 到 ######，對應 HTML 中的 h1 到 h6 標籤。

標題語法

標題的語法非常簡單，只需在文字前面加上對應數量的井號（#）符號，井號與文字之間需要有一個空格。標題的級別由井號的數量決定：

Markdown 語法

# 一級標題 (h1)
## 二級標題 (h2)
### 三級標題 (h3)
#### 四級標題 (h4)
##### 五級標題 (h5)
###### 六級標題 (h6)

複製

標題效果

一級標題 (h1)

二級標題 (h2)

三級標題 (h3)

四級標題 (h4)

五級標題 (h5)

六級標題 (h6)

替代語法

Markdown 還提供了另一種方式來創建一級和二級標題，使用的是底線風格：

Markdown 語法

一級標題
=======

二級標題
-------

複製

說明

等號（=）用於一級標題，連字符（-）用於二級標題。

等號和連字符的數量不限，只要有一個以上即可。這種語法更接近傳統的文本標記方式，某些情況下可能更直觀。

標題的最佳實踐

  • 文檔結構層次：一般來說，每個文檔應該只有一個一級標題，作為文檔的主標題。
  • 標題間隔：為了提高可讀性，建議在標題前後添加空行。
  • 標題編號：Markdown 本身不支援自動標題編號，但某些處理器或擴展可能提供此功能。
  • 標題錨點：許多 Markdown 實現會自動為標題生成錨點，方便文檔內部導航。
  • 避免過深的標題層次：一般建議不要使用超過四級的標題層次，以保持文檔清晰。

標題在不同 Markdown 實現中的差異

  • 某些實現可能要求井號和標題文字之間必須有空格，而其他實現可能更寬鬆。
  • 標題錨點的生成規則可能不同，有些會保留大小寫，有些會全部轉為小寫，有些會將空格轉換為連字符。
  • 某些擴展可能支援在標題中使用其他 Markdown 語法，如斜體、粗體或鏈接。
結論：標題是 Markdown 文檔結構的基石，掌握標題語法可以幫助您創建組織良好、層次分明的文檔。

字體大小與效果

✅ 原始 Markdown 標準功能 ⚠️ 部分平台擴展功能

本章涵蓋原始 Markdown 標準功能（粗體、斜體）和 GFM 擴展功能（刪除線）。上標、下標和高亮是特定平台擴展，兼容性有限。

在 Markdown 中，雖然沒有直接控制字體大小的語法，但可以通過標題級別間接控制文字大小。而對於字體效果，Markdown 提供了多種格式化選項，包括粗體、斜體、刪除線等，讓您的文字更具表現力和層次感。

字體大小

Markdown 本身不提供直接設置字體大小的語法，因為它的設計理念是專注於內容而非樣式。如果需要不同大小的文字，通常有以下幾種方法：

  • 使用標題級別：使用不同級別的標題可以產生不同大小的文字。
  • 使用 HTML 標籤：在大多數 Markdown 實現中，可以直接使用 HTML 標籤。
  • 使用 CSS：若 Markdown 會轉成 HTML 並支援自定義 CSS，可定義類別來控制字體大小。

HTML 標籤範例

<span style="font-size: 24px;">這是較大的文字</span>
<span style="font-size: 12px;">這是較小的文字</span>

CSS 類別範例

<span class="large-text">這是較大的文字</span>

.large-text {
  font-size: 24px;
}
需要注意的是，直接控制字體大小通常不符合 Markdown 的簡潔理念，而且可能在不同的 Markdown 渲染器中表現不一致。建議優先使用標題級別來區分內容的重要性和層次。

字體效果

Markdown 提供了多種基本的字體效果語法，讓您能夠強調文字或改變文字的呈現方式。

粗體

Markdown 語法

**這是粗體文字**
__這也是粗體文字__

複製

效果如下

這是粗體文字

這也是粗體文字

斜體

Markdown 語法

*這是斜體文字*
_這也是斜體文字_

效果如下

這是斜體文字

這也是斜體文字

粗斜體

Markdown 語法

***這是粗斜體文字***
___這也是粗斜體文字___
**_這也是粗斜體文字_**
__*這也是粗斜體文字*__

效果如下

這是粗斜體文字

這也是粗斜體文字

這也是粗斜體文字

這也是粗斜體文字

刪除線

GFM 擴展功能：刪除線是 GitHub Flavored Markdown 的擴展功能，原始 Markdown 不支援此語法。

Markdown 語法

~~這是帶有刪除線的文字~~

效果如下

這是帶有刪除線的文字

下劃線

Markdown 原生語法不支援下劃線，但可以使用 HTML 標籤：

HTML 語法

<u>這是帶有下劃線的文字</u>

效果如下

這是帶有下劃線的文字

上標和下標

特定平台擴展：此語法僅在特定實現中支援，不是標準 Markdown 或 GFM 的一部分。建議使用 HTML 標籤以確保兼容性。

擴展語法與 HTML 語法

上標：X^2^
下標：H~2~O

上標：X<sup>2</sup>
下標：H<sub>2</sub>O

效果如下

上標：X2

下標：H2O

高亮 / 標記

非標準語法：==高亮== 不屬於 GFM 標準，僅在部分編輯器中支援。建議使用 HTML 標籤。

擴展語法與 HTML 語法

==這是高亮文字==

<mark>這是高亮文字</mark>

效果如下

這是高亮文字

組合使用字體效果

Markdown 語法

**這是粗體文字，其中包含*斜體*部分**
*這是斜體文字，其中包含**粗體**部分*
~~這是刪除線文字，其中包含**粗體**部分~~

效果如下

這是粗體文字，其中包含斜體部分

這是斜體文字，其中包含粗體部分

這是刪除線文字，其中包含粗體部分

字體效果的最佳實踐

  • 適度使用：過度使用強調效果會降低其效果，甚至可能使文檔難以閱讀。
  • 一致性：在整個文檔中保持一致的強調風格。
  • 避免嵌套過深：過深的嵌套可能導致可讀性問題。
  • 空格處理：在某些 Markdown 實現中，強調標記周圍的空格可能影響渲染結果。

不同 Markdown 實現中的差異

  • 某些實現可能不支援所有上述效果，特別是上標、下標和高亮。
  • 某些實現可能有額外的字體效果語法。
  • HTML 標籤的支援程度也可能不同。
結論：雖然 Markdown 不直接提供字體大小控制，但通過標題級別和 HTML/CSS 可以實現類似效果。對於字體效果，Markdown 提供了豐富的選項，包括粗體、斜體、刪除線等，讓您能夠強調文字並創建層次分明的內容。

掌握這些字體效果語法，可以幫助您創建更具表現力和專業性的 Markdown 文檔，無論是技術文檔、部落格文章還是其他類型的內容。
