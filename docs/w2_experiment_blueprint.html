W2 實驗藍圖

主線 80% 探索 20% A3 Drift + Block Risk

W2 核心目標：固定 A3 例行回歸節奏，並把 block-level 風險比較做成可重跑報告，支援週會 go/no-go 判斷。

1. W2 目標與交付物

工作流 目標 W2 交付物
A3 例行回歸 + 漂移偵測 確認主線是否穩定，檢查相對 W1 是否發生漂移 A3 drift summary
block-level 穩健性對比 比較 B1/B2/B3 在不同 block 的 gate 表現 block risk report v1
W2 週結論 輸出主線節奏與 Event line 是否可重啟 W2 decision input

3. 執行流程

3.1 A3 例行回歸

./venv/bin/python -m simulation.track_a_protocol_regression

3.2 W2 block risk 報告生成

./venv/bin/python scripts/w2_block_risk_report_v1.py

3.3 漂移偵測規則（W1 -> W2）

以 W1 D5 mainline 為 reference，A3 最新資料為 current：

檢查項 門檻
a3_overall_passtrue
|delta_l1|<= 1
|delta_healthy|<= 3
|delta_mean_s3|<= 0.02

全部通過：stable；否則：drift_detected。

3.4 block-level 穩健性對比規則

以每個 block baseline 為對照，檢查 event family gate 結果：

  1. overall_pass
  2. new_l1
  3. l1 / healthy
  4. delta_mean_s3_vs_baseline
  5. fairness_fail_count

焦點比較：b1_async_poisson_r008 跨 4 個 blocks 的通過率與 new_l1。
