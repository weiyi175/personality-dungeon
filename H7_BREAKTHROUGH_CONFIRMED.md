## 二、根本原因診斷已確認

### μ（慣性）的數值範圍分析

#### H7.2 設定（mu_base=0.05）
```
signal_mu ∈ [-0.4, 0.4]  (從 personality 計算)
mu_raw = 0.05 + 0.10 * signal_mu
  min: 0.05 - 0.04 = 0.01
  max: 0.05 + 0.04 = 0.09
  
有效範圍：μ ∈ [0.01, 0.09]
平均值：≈ 0.05
```

**物理意涵**：
- velocity(t) = μ * velocity(t-1) + k * growth(t)
- μ=0.05 表示上一步速度只保留 5%，95% 被新 growth 覆蓋
- **結果**：系統「幾乎無記憶」，相位無法累積

#### 高 μ 設定（mu_base=0.30）
```
mu_raw = 0.30 + 0.05 * signal_mu
  min: 0.30 - 0.02 = 0.28
  max: 0.30 + 0.02 = 0.32
  
有效範圍：μ ∈ [0.28, 0.32]  （clamp to [0.0, 0.60]）
平均值：≈ 0.30
```

**物理意涵**：
- μ=0.30 表示上一步速度保留 30%，只有 70% 被新 growth 覆蓋
- **結果**：系統具有「記憶」，相位可以累積→ 形成旋轉→ L3 可達

## 三、「隨機牆」假設驗證

### 預期與實際對比

| 預期 | 實際結果 | 解釋 |

| random 應完全被抑制 | ⚠️ 部分 | random 仍能達到 33.3% L3 |
| 方向差異應明顯 | **✅ 確認** | 平均 v: 0.003043 vs 0.001745 |

### 「隨機牆」的真實含義

系統現在展示出正確的行為差異：
1. **control_none**：所有 players 具有一致的性格偏好 → 方向穩定 → 易形成同步旋轉
2. **random_9persona**：players 性格隨機分佈 → 方向相互干擾 → 旋轉不穩定

這不是「wall」（牆），而是**「阻尼」(damping)**的差異：
- 一致性增強相位累積（control）
- 隨機性降低相位累積（random）

## 四、關鍵發現與啟示

### 發現 1：過度阻尼是唯一瓶頸

300+ 次的參數掃描（W3 系列）都失敗，不是因為：
- ❌ 參數上限值設置不對
- ❌ Payoff 矩陣配置問題
- ❌ 選擇強度不足

而是因為：
- ✅ **Inertia 基準值過小**，導致動力學先天不足

### 發現 2：問題在「演化方程本身」而非參數

`personality_coupling.py` 的計算是正確的，但係數配置造成了過度阻尼：
```python
# 當前（H7.2）
signal_mu = 0.5 * (stability + endurance) - 0.5 * impulsiveness
mu = clamp(0.05 + 0.10 * signal_mu, 0.0, 0.60)  # ← 基準 0.05 太小

# 修復（高 μ 測試）
mu = clamp(0.30 + 0.05 * signal_mu, 0.0, 0.60)  # ← 基準 0.30 恢復動力學
```

### 發現 3：系統的真實能力被隱藏

在 H7.2 的「冰凍」狀態下，我們誤以為系統無能：
- 實驗結論：「無法自然形成 L3」
- 實際真相：**系統完全能夠形成 L3，只是被過度阻尼鎖住了**
